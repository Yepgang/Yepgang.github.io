Posted on 2020-11-30 | In 电信数据分析案例

pro2：java项目的构建和idea的适应

  • 关于Object：Object 类位于 java.lang 包中，编译时会自动导入，我们创建一个类时，如果没有明确继承一个父类，那么它就会自动继承 Object，成为 Object 的子类
  • 关于interface：
    • 抽象方法的集合，不包含方法的实现，默认属性public abstract
    • 要求继承类实现所有的方法(抽象类除外)
    • 不用于实例化，没有构造函数
    • 类不支持多继承，接口支持
    • 成员变量默认属性为public static final
      • 关于final
        • 修饰常量：该常量不可修改，需要当场赋值
        • 修饰方法：不可重写
        • 修饰类：无法被继承
    • 接口可以继承接口
    • 接口和接口内的方法都是隐式抽象
  • override关键字：主打一个检错的效果，方便阅读
  • closeable：提供close方法
  • 关于泛型：public List read(Class clazz) throws IOException;
    • 泛型方法：放在返回值之前，可以用extend限制E的上界
      • 泛型参数/泛型返回值
    • 泛型类：放在类名之后
    • 类型通配符：List<?>,super关键字限制下界
    • Class clazz中，clazz.newInstance()返回一个T的实例
  • Data类和SimpleDateFormat常用来处理日期和时间：
    • sdf:parse(string)返回data，format(data)返回string，sdf(format)充当桥梁
  • NumberFormat和DecimalFormat常用来格式化数字
  • string、stringbuilder、stringbuffer
    • String：适用于少量的字符串操作的情况
    • StringBuilder：适用于单线程下在字符缓冲区进行大量操作的情况
    • StringBuffer：适用多线程下在字符缓冲区进行大量操作的情况
    • String生成常量对象，其他生成变量对象
  • 关于tostring：当我们打印一个对象的引用/做连接操作时，默认调用这个对象的tostring
  • 有时间学学java io(…)
  • 之后补下java高级篇的内容(volitale,thread)
  • properties
  • CalllogConsumer.class.getClassLoader().getResourceAsStream()
  • KafkaConsumer<String, String>
  • ConsumerRecords

java基础学习

Posted on 2020-11-29 | In java

基础易忘点

  • main方法时Java程序的入口，写法固定(为什么?)
  • 源文件.java,与public类同名，对应唯一一个public类(why?)
  • javadoc:API文档(待学习)
  • 关键字/保留字/标识符(严格区分大小写)
  • 包名：xxxyyyzzz; 类/接口：XxxYyyZzz; 变量/方法：xxxYyyZzz 常量：XXX_YYY_ZZZ
  • JRE：jvm + 库；JDK: JRE + 工具
  • Java各整数类型/浮点类型有固定的表数范围和字段长度，不受具体OS的影响，以保证java程序的可移植性
  • Java的浮点型常量默认为double型 整型默认为int
  • Java中的所有字符都使用Unicode编码(UTF-8)，char(2 byte)
  • boolean类型数据只允许取值true和false，无null,不可以使用0或非 0 的整数替代false和true(所以不参与计算),在编译之后都使用java虚拟机中的int数据类型来代替
  • byte,short,char之间不会相互转换，他们三者在计算时首先转换为int类型(容量小的类型自动转换为容量大的数据类型)
  • 基本数据类型/引用数据类型(string)；String同化之王；String str1 = 4；(false)
  • 强制类型转换：boolean类型不可以转换为其它的数据类型
  • 计算机底层都以补码的方式来存储数据
  • instanceof: 返回boolen
  • “&”和“&&”的区别；“|”和“||”的区别；一般用后者；
  • e.g. >>>:无符号右移
  • return直接结束整个方法
  • 声明数组不指定长度 int[] a / int a[]
  • 初始化(多维)数组：动态/静态 多维数组不一定时规则矩阵
  • boolen默认为false；引用默认为null

面向对象

  • 匿名对象：不设置对象的句柄，直接调用方法
  • overload：参数不同；可变数量的参数；
  • 参数的值传递机制(基本/引用数据类型)
  • 构造器：隐式/显示(可以overload)，无返回类型
  • “this(形参列表)”：必须声明在类的构造器的首行
  • package->import->class
  • java只支持单层/多层继承(extend),子类无法访问父类private
  • overwrite
  • 子类中所有的构造器默认都会访问父类中空参数的构造器
  • 当父类中没有空参数的构造器时，子类的构造器必须通过this(参数列表)或者super(参数列表)语句指定调用本类或者父类中相应的构造器。同时，只能”二选一”，且必须放在构造器的首行
  • 对象的多态性：父类的引用指向子类的对象->upcasting; 该引用无法访问子类添加的属性和方法
  • 重载是静态绑定(编译)；多态是动态绑定(运行期间)；成员变量不具有多态性
  • static：修饰的成员高于对象，所有对象共享，和类一起加载；static方法只能访问类的static修饰的属性或方法(内部肯定没this…),无法被重写；可以用类名.方法名()的形式访问由static修饰的类方法
  • 代码块(待定…)
  • final：类、方法、变量；无法继承、重写、改变(const)
  • abstract:类、方法；只能继承、只能重写
  • interface：弥补多重继承；接口不能继承抽象类，但允许继承多个接口
  • 内部类(待定…)

异常处理

  • try…catch…;throws(声明抛出异常);throw(不太会);
  • 这一块实战中体验；

今后要做什么工作呢？

Posted on 2020-11-28 | In 杂谈

研究生已经开学两个月了，时间飞快呢。开学这段时间，我花了很多时间思考关于工作的事情，今后到底要从事一份什么工作呢？

周围的大部分人都想着做机器学习、算法这些岗位，我自己其实本科也有所涉猎，不是很有感觉。刚好最近在上大数据相关的课程，就觉得以后做大数据开发还挺不错，决定朝这方面努力。

一开始学大数据，大家都是从hadoop这个生态入手，它的生态圈很扎实，内容不少。我是从Hadoop权威指南这本书入门，学习了它的前五章。之后觉得还是得动手，“纸上得来终觉浅”，那我该如何上手呢？

感谢互联网时代让知识的获取如此快捷有效。在一定的探索后，我锁定了两个教程，一个是林子雨老师出的教程，一个是尚硅谷大数据系列的教程。前者很适合大数据框架的基础入门，后者更加细致，涵盖大数据相关知识的方方面面。

我的学习思路是这样的。首先跟着林子雨老师快速过一遍hadoop生态系统的安装和基本使用，在这期间需要熟悉linux的使用，而且之后的开发环境都是使用linux；除此之外还要学习java，之前也没有学过，这里就跟着尚硅谷的java教程学习了java基础。

在熟悉了以上最基本的内容后，我的动手之路显然不够，我决定开始做一个完整的大数据项目，这里我选的是尚硅谷出的电信客服案例。它里面有很多新的知识需要我学习，我打算边做边学，这也是最近在忙的内容。

初步想法

我想做大数据相关的研究，初步考虑的是以下两个方向。

  • 考虑到大数据技术体系目前已经趋于成熟，从技术体系上寻求创新和突破还是具有一定难度的，但是由于目前大数据相关技术尚处在落地应用的初期，所以有大量与行业落地相结合的创新点
  • 另外一个切入点是大数据分析。作为数据价值化的重要手段，大数据分析与机器学习有紧密的联系，所以大数据与机器学习相结合也是一个不错的选择

工作开展

  • 阅读感兴趣的论文
  • 阅读大数据领域的经典论文

    • Big data: Dimensions, evolution, impacts, and challenges
      • 本文介绍了大数据的集成视图，跟踪了大数据在过去20年中的发展，并讨论了处理各种结构化和非结构化数据必不可少的数据分析
