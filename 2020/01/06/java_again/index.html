概述

之前学习java，只学习了Java最基本的内容，最近准备从以下两个方面扎实地学习下java：

  • 更细致地学习java知识本身，更多地是记录学习过程中的思考而非知识
  • 借机回顾下算法和数据结构相关地知识，语言当然是用java，考虑到最近的代码大概是leetcode级别，ide我还是用vscode

chapter1 基础问题

  • JRE：jvm + javaSE标准类库；JDK: JRE + 工具(javac/java)
  • main方法时Java程序的入口，写法固定
    • jvm运行程序时调用main方法，所以需要定义为public
    • 此时并没有创建包含main方法的类，所以需要定义为static
    • main方法没有返回值，所以声明为void
      • java是针对多线程设计的语言，main的完成并不意味整个程序都完成，所以main的返回类型没有意义
      • System.exit(int),退出程序
        • 0(normal);1(exception);-1(error)
    • main方法接受命令行的多个参数，所以需要参数args数组
    • main方法可以重载、调用、继承、重写、抛出异常、带有类型参数(??)
  • 源文件.java,与public类同名，对应唯一一个public类
    • public类作为文件对外的接口，只能有唯一一个
    • public类与文件同名可以加快jvm查找类的速度，编译更快
  • javadoc:API文档https://docs.oracle.com/en/java/javase/15/docs/api/index.html
    • 进一步阅读java源码(这个等之后看有没有必要)
  • 关键字/保留字/标识符(严格区分大小写)
    • 包名：xxxyyyzzz; 类/接口：XxxYyyZzz; 变量/方法：xxxYyyZzz 常量：XXX_YYY_ZZZ
    • 代码简洁知道：无重复代码、提高表达力、简单抽象
  • Java各整数类型/浮点类型有固定的表数范围和字段长度，不受具体OS的影响，以保证java程序的可移植性
    • Java的浮点型常量默认为double型 整型默认为int
    • Java中的所有字符都使用Unicode字符集(UTF-8)，char(2 byte)
      • unicode字符集：包含所有符号的字符集；utf-8:实现unicode的一种变长编码方式
  • boolean类型数据在java中是单独的数据类型，只允许取值true和false，默认为false，boolean类型不可以转换为其它的数据类型
    • 不可以使用0或非 0 的整数替代false和true(所以不参与计算),在编译之后都使用java虚拟机中的int数据类型来代替
  • 关于null
    • 大小写敏感
    • 只是一个特殊值，并非对象/类型，也不是任何对象的实例，可以赋值给任何引用类型,不能赋值给基础类型
    • 引用变量的默认值为null
    • null主要用来临时标识不确定的对象，可以调用静态方法
  • byte,short,char之间不会相互转换，他们三者在计算时首先转换为int类型(容量小的类型自动转换为容量大的数据类型)
  • 基本数据类型/引用数据类型(包括数组)
    • 基本数据类型：main栈；引用数据类型：堆
    • 引用数据类型的值为对象的地址
    • ==
      • 基本数据类型：比较值
      • 引用数据类型：比较存放的地址值
      • 如果想比较内容，一般有equal方法
    • string对象本身不可改变，但string对象的引用是可以指向新的对象的
  • instanceof: 返回boolen
  • “&”和“&&”的区别；“|”和“||”的区别
    • 后者短路计算，速度更快
  • e.g. >>>:无符号右移
  • 数组：声明、分配空间、赋值
    • 声明数组不指定长度 int[] a / int a[]；分配空间用new(分配空间的默认值是0)；赋值：静态/动态
  • return直接结束整个方法
  • 初始化(多维)数组：动态/静态 多维数组不一定时规则矩阵

chapter2 面向对象

  • 好处
    • 避免对数据进行不合理的操作
    • 代码更简洁
    • 方便代码的使用和维护
    • 方便代码功能的拓展

      part1 类和对象

  • 类/对象(实例)
  • 内存解析
    • 栈：局部变量，对象引用
    • 堆：对象实例，数组
    • 方法区：类信息 、常量 、静态变量/方法
  • 匿名对象：不设置对象的句柄，直接调用方法
    • 只调用一次方法
    • 将匿名对象作为参数
  • overload：方法的重载，需要参数不同；
  • …表示可变数量的参数；
  • 参数的值传递机制(基本/引用数据类型)
    • 基本数据类型：传递值副本
    • 引用数据类型：传递地址

      part2 封装

  • 好处(个人理解)
    • 避免对数据进行不合理的操作
    • 方便代码的使用和维护
  • 实现途径
    • 访问权限修饰符
    • 构造器：隐式/显式(可以overload)，无返回类型，与对象同名
      • 作用是初始化对象
      • this关键字初始化对象时，”this(形参列表)”：必须声明在类的构造器的首行
    • package->import->class
      • package方便管理，也是封装的体现
      • import告诉编译器去哪里寻找类

        part3 继承

  • 好处
    • 代码更简洁
    • 方便了代码功能的拓展
    • 多态的前提
  • 实现
    • extend
      • java只支持单层/多层继承(extend),子类无法访问父类private
    • overwrite
      • 关于重写的细节理解有限(…)
    • 构造器
      • this/super
      • 子类中所有的构造器默认都会访问父类中空参数的构造器
      • 当父类中没有空参数的构造器时，子类的构造器必须通过this(参数列表)或者super(参数列表)语句指定调用本类或者父类中相应的构造器。同时，只能”二选一”，且必须放在构造器的首行
        • 二选一是避免重复构造
        • 放在首行是为了避免操作还未初始化的对象

          part4 多态

  • 好处
    • 方便代码功能的拓展
  • 实现
    • 编译期间多态：重载，对应静态绑定
    • 运行期间多态：重写，对应动态绑定
    • 多态针对的是方法而不是成员变量
    • 运行期间多态的实现：父类的引用指向子类的对象；该引用无法访问子类添加的属性和方法
  • object类/包装类
    • 基本数据类型-包装类(父类是number)-string类
    • 现在有自动装箱/插箱，使用起来很方便，但是要注意尽量不要生成大量的无用对象

part5 在三大特性的基础上对类的一些补充

  • static
    • 作用：修饰所有对象共享的变量/方法，仅仅通过类本身调用
    • static随类一起加载，static方法只能访问类的static修饰的属性或方法(内部肯定没this…),
    • 可以被重写，但是实现不了多态，因为静态方法会在编译期间被绑定
  • 代码块
    • 作用：初始化呗，做部分构造器的工作
    • 只能被static或非static修饰
      • 静态代码块
      • 非静态代码块，对象创建时执行一次，优于构造器
  • final
    • 类、方法、变量；无法继承、重写、改变(const)
  • abstract
    • 类、方法；只能继承、只能重写，不能实例化抽象类，这是abstract的作用
  • interface：
    • 作用：制定规范，public修饰
    • 抽象/静态/default方法和常量的集合：方法默认public abstract，常量默认public static final
      • 默认方法(…)
    • 弥补多重继承；接口不能继承类，只能继承接口
  • 内部类
    • 作用：内部类可以继承单独的接口/类，更好的实现多继承效果
    • 成员/局部内部类/匿名内部类：依附于Outer Class的实例，即隐含地持有Outer.this实例，并拥有Outer Class的private访问权限
    • 静态内部类：独立类，但拥有Outer Class的private访问权限

chapter 3 高级特性

多线程

  • 基本概念：程序、进程、线程
    • 程序：一组指令的集合，用于完成特定的任务
    • 进程：正在运行的一个程序，存在生命周期；是资源分配的基本单位，对应单独的内存区域
    • 线程：进程可进一步划分为线程，一个进程可以并行执行多个线程；是调度和执行的基本单位，对应独立的运行栈和程序计数器
      • 优点：改善程序结构，各个线程独立运行，利于修改；线程切换的开销小；同一进程的多个线程共享相同的内存空间,线程间的通信更简便
      • 缺点：多个线程操作共享的系统资源会带来安全隐患，需要有同步机制
    • e.g.一个java程序至少包括三个线程：main()主线程、gc()垃圾回收线程、异常处理线程
  • 线程的创建和启动
    • java.lang.Thread、start()调用run()
    • 创建有两种方法(还有Callable接口、线程池这里不展开了)
      • 继承Thread类，重写run方法
      • 实现Runnable接口，将其子类对象作为参数传递给Thread类的构造器中
        • 好处：避免了单继承的局限；多个线程可以共享同一个接口实现类的对象
  • 线程的调度
    • 同优先级线程组成先进先出队列
    • 高优先级线程采用优先调度
    • 线程的优先级1-10，线程创建时继承父线程的优先级
  • 线程的分类
    • 守护线程/用户线程
    • 守护线程服务用户线程，若JVM中没有用户线程，当前JVM退出
  • 线程的生命周期
    • 新建、就绪、运行、阻塞、死亡
  • 同步机制
    • synchronized:同步代码块、方法
    • lock：显示锁，同步代码块，性能更好
    • 线程的通信
      • Synchronized的语义底层是通过一个monitor(监视器锁)的对象来完成，其实wait/notify等方法也依赖于monitor对象，wait()notify()notifyAll(),只能在synchronized方法/代码块中使用
  • 死锁：不同的线程分别占用对方需要步资源放弃，都在等待自己需要的资源，就形成了线程的死锁

java集合

  • 分为Collection和Map两种体系：)
  • Collection接口：包含可操作List/Set的方法
  • Iterator接口
    • iterator()，返回实现了Iterator接口的对象
    • 用于遍历集合，同时不暴露集合内部的细节
    • 每次调用iterator()得到一个全新的迭代器对象
    • hasNext->next
    • foreach循环底层用Iterator实现，遍历不需获取Collection的长度，也不需要索引
  • List
    • Collection子接口，添加了一些操作索引的方法，解决数组长度写死的局限性
    • 常用的实现类：ArrayList、LinkedList、Vector
      • ArrayList:数组实现的变长数组，方便随机访问
      • LinkedList:双向链表实现，适合插入和删除
      • Vector:与ArrayList类似，但是是强同步类，速度比ArrayList慢
      • Vector还有一个子类Stack
  • Set
    • Collection子接口，没有提供额外的方法
    • 常用实现类：HashSet、LinkedHashSet
      • HashSet:必须重写equals和hashCode方法
      • LinkedHashSet是HashSet的子类，在后者的基础上用双向链表维护元素的插入顺序，所以性能略低于后者
      • TreeSet：底层采用红黑树结构存储数据，自然排序/定制排序(…)
  • Map
    • 与Collection并列存在，保存key-vaule型数据
    • key用Set存放，同一个Map对象对应的类必须重写hashcode和equles方法
    • 常用实现类：HashMap、TreeMap、LinkedHashMap、Properties
    • HashMap:数组+链表+树实现
    • LinkedHashMap:HashMap的子类，在后者的基础上用双向链表维护元素的插入顺序，所以性能略低于后者
    • TreeMap(…)
    • HashTable:与HashMap类似，但是线程安全，很多情况下可以互用，需要注意的是HashTable不允许使用null作为key和vaule
    • Properties:HashMap的子类，适合处理配置文件
      • setProperty()/getProperty()
  • Collections工具类
    • 操作Set、List、Map的工具类
    • 提供排序、查找、替换、同步控制等大量静态方法

泛型/通配符

  • 泛型
    • 增加程序的可读性和稳定性
    • 可以将类或者方法中的参数类型化，增加代码复用性
  • 通配符
    • 在对泛型进行赋值时，当赋值的类型不确定的时候，我们用通配符(?)代替，方便引用包含多种类型的泛型，进一步提高代码复用性
    • 部分方法会丢失原始集合中的类型信息，返回object对象，也没办法往里面添加非object对象
    • 可以配合extend/super使用：限定通配符

I/O

  • File类
    • 用于表示一个真实存在的文件或目录并对其进行新建、删除、重命名等等
    • File对象可能没有真实存在的文件或目录
    • 可以作为参数传递给流的构造器
    • 路径分隔符：/或者\或者File.separator
  • I/O用于处理设备之间的数据传输，分为字节流/字符流
  • System:in/out分别对应InputStream和PrintStream，后者是OutputStream的子类
    • 可以通过setIn/setOut重定向
  • NIO：Path、Files…

反射

  • 本质：允许程序在运行期间借助反射机制获取类的内部信息并对其进行操作
    • 一个类对应JVM一个Class类型对象,即.class文件，包含类的所有被加载的结构
    • java.lang.Class、java.lang.relect.[Method,Filed,Constuctor]
    • 在Object类中定义了public final class getClass()
    • Class类是反射的本质
    • 获取方式
      • .class、.getClass、class.forName()
  • 类的加载
    • 类加载器：将class文件读入内存，创建class对象
    • 链接：将类的二进制数据合并到JRE中
    • Jvm：初始化，执行类构造器方法
