java基础学习

Posted on 2020-11-29 | In java

基础易忘点

  • main方法时Java程序的入口，写法固定(为什么?)
  • 源文件.java,与public类同名，对应唯一一个public类(why?)
  • javadoc:API文档(待学习)
  • 关键字/保留字/标识符(严格区分大小写)
  • 包名：xxxyyyzzz; 类/接口：XxxYyyZzz; 变量/方法：xxxYyyZzz 常量：XXX_YYY_ZZZ
  • JRE：jvm + 库；JDK: JRE + 工具
  • Java各整数类型/浮点类型有固定的表数范围和字段长度，不受具体OS的影响，以保证java程序的可移植性
  • Java的浮点型常量默认为double型 整型默认为int
  • Java中的所有字符都使用Unicode编码(UTF-8)，char(2 byte)
  • boolean类型数据只允许取值true和false，无null,不可以使用0或非 0 的整数替代false和true(所以不参与计算),在编译之后都使用java虚拟机中的int数据类型来代替
  • byte,short,char之间不会相互转换，他们三者在计算时首先转换为int类型(容量小的类型自动转换为容量大的数据类型)
  • 基本数据类型/引用数据类型(string)；String同化之王；String str1 = 4；(false)
  • 强制类型转换：boolean类型不可以转换为其它的数据类型
  • 计算机底层都以补码的方式来存储数据
  • instanceof: 返回boolen
  • “&”和“&&”的区别；“|”和“||”的区别；一般用后者；
  • e.g. >>>:无符号右移
  • return直接结束整个方法
  • 声明数组不指定长度 int[] a / int a[]
  • 初始化(多维)数组：动态/静态 多维数组不一定时规则矩阵
  • boolen默认为false；引用默认为null

面向对象

  • 匿名对象：不设置对象的句柄，直接调用方法
  • overload：参数不同；可变数量的参数；
  • 参数的值传递机制(基本/引用数据类型)
  • 构造器：隐式/显示(可以overload)，无返回类型
  • “this(形参列表)”：必须声明在类的构造器的首行
  • package->import->class
  • java只支持单层/多层继承(extend),子类无法访问父类private
  • overwrite
  • 子类中所有的构造器默认都会访问父类中空参数的构造器
  • 当父类中没有空参数的构造器时，子类的构造器必须通过this(参数列表)或者super(参数列表)语句指定调用本类或者父类中相应的构造器。同时，只能”二选一”，且必须放在构造器的首行
  • 对象的多态性：父类的引用指向子类的对象->upcasting; 该引用无法访问子类添加的属性和方法
  • 重载是静态绑定(编译)；多态是动态绑定(运行期间)；成员变量不具有多态性
  • static：修饰的成员高于对象，所有对象共享，和类一起加载；static方法只能访问类的static修饰的属性或方法(内部肯定没this…),无法被重写；可以用类名.方法名()的形式访问由static修饰的类方法
  • 代码块(待定…)
  • final：类、方法、变量；无法继承、重写、改变(const)
  • abstract:类、方法；只能继承、只能重写
  • interface：弥补多重继承；接口不能继承抽象类，但允许继承多个接口
  • 内部类(待定…)

异常处理

  • try…catch…;throws(声明抛出异常);throw(不太会);
  • 这一块实战中体验；
