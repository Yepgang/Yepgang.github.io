lecture 1 Introduction and Strassen’s Algorithm

  • Asymptotic complexity
    • 渐进复杂度，大O(<=)、小o(<)、Ω(>=)、Θ(=),w(>)
    • 一般我们分析算法更关心Θ而不是上界
    • 通过求极限确定渐进符号，一个有用的结论$lim_{n\to\infty}\frac{(log n)^k}{n}=0$
  • Strassen’s Alogirthm
    • 第一个时间复杂度低于$O(n^3)$的矩阵乘法算法,用来介绍时间复杂度还真不错
    • 对比
      • 常规计算和简单分块递归的时间复杂度均为$O(n^3)$，空间复杂度$O(n^2)$
      • Strassen’s Alogirthm的时间复杂度为$O(n^{log_2 7})
        • tree of recursive calls：常用来计算递归的复杂度
      • 对于普通的递归，有8个小矩阵的运算而strassen只有7个，所以后者更快
    • 特性分析
      • 当n>10000时该算法效果才比常规计算快，所以对于递归后产生的小规模矩阵可以采取常规计算加快速度
      • 空间复杂度
        • 如果计算s1-s7使用不同的空间，递归计算出来的空间复杂度是$O(n^{log_2 7})$，比$O(n^2)$大
        • 使用相同的空间的空间复杂度是$O(n^2)$
  • Recurrences and how to solve them
    • 两个基本问题
      • 递归到较小规模后如何处理：采取常量级别的其他简单方法
      • 当n不满足一定的条件怎么办？(比如merge sort中n不为2的幂)：一般来说我们不用关心这个，我们只用考虑满足条件的n，如果n不满足条件，我们可以证明处理规模为n的问题时间复杂度与所求的通用的时间复杂度是常数量接近的关系
    • 几种常见的方法
      • unrolling：展开，简单直观
      • guess and inductive proof:猜想，归纳证明，很数学
      • recursion tree：好用
        • 一个简单的推导$a^{log_b n}=b^{(log_b a)(log_b n)}=n^{log_b a}$
      • master formula:这真是我见过最好的理解了

lecture 2 dynamic programming 1

  • basic idea
    • 常用于需要指数级算法解决问题的场景，将问题的时间复杂度降低到$O(n^2)/O(n^3)$
    • 类似分治法，将大问题变成多个子问题，不同点在于dp的子问题通常有重合的部分
    • 具体思路：将问题分解成一定量的子问题，针对每个子问题求它的最优解以得到更大问题的最优解
    • 根据思路，我们逆向推得：如果一个问题可以分解成求子问题的最优解，那我们就可以考虑用dp策略了
  • Longest Common Subsequence
    • 简介：长度为m、n的两个序列
    • 问题包括两部分：lcs的长度/lcs本身，对于后者逆向分析即可
    • 引出两种动态规划，试图写出O(mn)的算法
      • bottom-up dp
      • top-down dp (memoized)
      • 其实差别不大，能写出直观的代码就行
    • extension
      • an equivalent problem:”minimum edit distance”,e.g diff command.
        • |S|−LCS(S, T) deletes and |T|−LCS(S, T)inserts
      • 很多涉及到序列的问题本质上都可以用dp作为解决方案
  • The Knapsack Problem
    • 简介：n个事件，对应单独的cost和vaule，总cost为S，求最大vaule，这里的cost是整数
    • 问题：最大结果/解决方案(逆向分析)
    • 两种：背包问题可/不可按比例得分
      • 前者：贪心算法
      • 后者：dp,本质上是决定背包事件有还是没有，即0/1两种状态，从而写递推公式V(k,B)
      • 策略：top-down dp (memoized)；复杂度：O(n*S)
    • extension
      • 如果存在cost为小数??
  • Optimal Binary Search Trees
    • 简介：二叉搜索树，节点有序，每个节点对应搜索频率，搜索都是成功搜索，求最优二叉搜索树
    • 本质：每个最优二叉搜索树的子树为最优二叉搜索树，选取不同的点作为根节点即可
    • 出于二叉搜索树的性质以及以上思路，令$c_{i,j}$表示$key_i$到$key_j$最优子结构产生的cost
    • 时间复杂度：$O(n^3)$
  • tree dp
    • 简介：对于一个树，选取点的子集使得点的权重之和最大且不存在两个点为某条边的两个端点
    • 思路：类似背包问题，本质上是点的0/1状态问题，不同的地方在于由于是树，所以考虑从根节点出发写递归公式

    • using dp on path-finding problems in graphs，这里是研究几个最短路径问题，根据源节点和目的节点的类型有所不同，针对的都是有向图(无向图没啥意义)

  • Dijkstra’s algorithm
    • 针对单源最短路径
    • 限制：边为正权重
    • 证明方法：数学归纳法
    • 时间复杂度
      • 邻接表：O($n^2+m$)，适用于稀疏图
      • 普通堆：O($(m+n)logn$),每条边处理一次，每次入堆一次(m),每个点出堆一次(n)
      • 斐波拉契堆：O(m+nlogn)(…)
  • The Bellman-Ford Algorithm
    • 对比Dijkstra’s algorithm
      • 都是对边做松弛操作，Dijkstra’s以贪心法选取未被处理的具有最小权值的节点，然后对其的出边进行松弛操作；而Bellman-Ford Algorithm简单地对所有边进行松弛操作
      • 可以适用于负权重的边,可以判断是否存在负权重的环
      • 时间复杂度O(mn)
    • 实现以及如何判断负权重的环
      • 常见思路：对所有的边进行(n-1)次松弛操作，再做一次松弛操作判断负环
      • dp：)
      • 找出负权重的环
        • 记录parent关系
        • 利用d[v] = d[x] +len(x, v)

    • 不再对边做松弛操作，而是对路径进行dp，处理过程与边的权值无关，因此可以处理负权重的边
    • dist_k(u, v) = length shortest path u → v using only points ≤ k,dist_n(u, v)为最终结果
    • 对所有的disk_n(u,u)作非负检测即可判断是否有负环
    • Floyd-Warshall runs in time Θ($n^3$)and space Θ($n^2$)

lecture 5 Amortized Analysis

  • 用处
    • The motivation for amortized analysis is that looking at the worst-case time per operation can be too pessimistic if the only way to produce an expensive operation is to “set it up” with a large number of cheap operations beforehand
    • 简而言之，如果产生一个复杂的操作之前需要一系列简单的操作，我们可以均摊分析这些操作的耗时(一般是常数量)，而不是单纯只分析最坏的情况，这里的均摊分析并不是针对输入做概率上的平均分析，而是对上述的多次操作做均摊分析
  • 均摊分析的三种方式，根据需要使用
    • 聚类分析：所有操作的耗时除以操作的次数，比较直接
    • 银行家分析：类似于记账，对不同的操作做记录，多存少补
    • 势能函数分析：将银行家分析具体化，提出势能函数
  • 聚类分析

lecture 6 Splay trees

  • a binary search tree (BST)
    • 对于bst，如果它相对比较平衡，我们搜索一个元素的复杂度是O(log n),但是如果它不够平衡，复杂度就变为O(n),所以我们要考虑的就是如何以较低的时间复杂度平衡bst
    • 如果插入的序列是随机的，bst的平均高度是O(log n),效果其实是很好的
      • 先求en的递推式，然后数学归纳法
  • 关于维持平衡的两个角度
    • 动态方法：将访问到的元素x放到一个新的位置方便下次访问
    • 静态方法：根据访问概率静态存放所有的元素
    • 实例：访问list
      • move-to-front lists (MTF): after an access to x, move x to the front of the list
      • compute access frequencies and sort the list accordingly
      • 结论：对于MTF，we have an amortized cost twice the cost in the optimal static list
  • 基于动态的角度，引入Splay trees
    • 有自己特有的rotation，每次将访问的节点转移到根节点处。这是splay tree的本质特征
    • 均摊分析)
    • 结论)
    • 对于插入，合并，删除，均摊复杂度均为O(log n)级别
      • 插入：先查找再插入；合并：先splay再合并；删除：先splay到根节点然后删除合并
